Claude Code CLI Works now

Claude Code (the terminal CLI) reads CLAUDE.md automatically from your project root on every session. One command installs it:

terminal
bash <(curl -fsSL https://raw.githubusercontent.com/Larens94/codedna/main/integrations/install.sh) claude

This copies CLAUDE.md to your repo root. Claude Code loads it automatically — no further configuration needed.

What the CLAUDE.md protocol enables

What you getHow
Annotate all files in a projectcodedna init ./ via CLI
Coverage reportcodedna check ./ — unannotated files and stale used_by:
Architectural mapcodedna manifest ./ — full package map in one pass
Cross-session agent memoryagent: + message: fields in every file
Cascade dependency awarenessused_by: graph + [cascade] tags
Claude Code has no plugin marketplace — CLAUDE.md is the standard integration method, used by all major AI coding tools (Cursor, Windsurf, Cline use equivalent rule files).

CLI Reference

Annotate any project from the terminal. Supports local models via Ollama at zero cost.

Install

# All providers — Anthropic, OpenAI, Gemini, Ollama
pip install 'codedna[litellm]'

# Anthropic only (legacy)
pip install 'codedna[anthropic]'

# No AI — structural annotations only, free
pip install codedna

Commands

CommandWhat it does
codedna init PATHFirst-time annotation — L1 module headers + L2 function Rules:
codedna update PATHIncremental — only unannotated files. Safe to re-run.
codedna check PATHCoverage report. Exits 0 if fully annotated.

Examples

# Free — local model via Ollama
codedna init ./ --model ollama/llama3

# Free — structural only, no AI
codedna init ./ --no-llm

# Preview without writing files
codedna init ./ --dry-run

# Anthropic Haiku
ANTHROPIC_API_KEY=sk-... codedna init ./ --model claude-haiku-4-5-20251001

# Coverage check
codedna check ./ -v
