Limite riscontrato

Parser TypeScript — gap noti

Il pass strutturale --no-llm via tree-sitter non riconosce tre costrutti TypeScript comuni. Di conseguenza 83 file TSX sono stati inizialmente annotati con exports: none pur avendo export reali.

Nota
Il supporto a TypeScript/TSX è ancora in fase sperimentale come dichiarato nella documentazione ufficiale CodeDNA. Il parser tree-sitter non copre tutte le sintassi moderne di React/Inertia: per questo nel nostro progetto (frontend quasi interamente .tsx) è stato necessario un secondo giro via LLM per recuperare gli export mancanti.
SintassiEsempioEstratto?
Aggregate export export { Button, buttonVariants } perso
Default export nominato export default function Login() perso
Default arrow anonimo export default ({ children }) => ... atteso
Named function/const export export function formatCurrency() ok

Workaround adottato: pass Fase 2 via LLM che "vede" correttamente la sintassi. Coerente con lo stato sperimentale di TSX. Il bug è candidato per un'issue upstream su github.com/Larens94/codedna.

Regole architetturali catturate

Il pass LLM ha estratto invariant non ovvi che il CLI strutturale non avrebbe mai scoperto. Sono ora in rules: e visibili al prossimo agente.

Season.phpModel
is_active deve avere un solo true — l'accessor con cache diventa non-deterministico altrimenti.
Payment.phpModel
payment_intent_id è l'idempotency key per retry webhook Stripe/PayPal. Non rigenerare mai.
EventRoom.phpModel
min_capacity / max_capacity override Room.max_capacity (rif. commit c39ad6a, b39dc11).
HandleInertiaRequestsMiddleware
share() è il contratto verso il frontend — le chiavi devono corrispondere a SharedProps in resources/js/types/index.d.ts.
CartController.phpController
La session key room_reservations è condivisa con il middleware Inertia — modificare qui richiede auditare il middleware.
User.phpModel
password usa il cast hashed di Laravel 11 — mai assegnare un valore pre-hashed.
routes/web.phpRoutes
I route names sono public API consumata da ziggy-js nel frontend — rinominare rompe tutti i route('name') in .tsx.
