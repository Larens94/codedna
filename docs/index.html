Architecture

Two levels. Zero context drift.

Traditional approaches treat context as something to fetch. CodeDNA treats it as something to embed — a navigation protocol baked into every file, at every level.

1

Macro-Context — ~60 tokens per file

The very first thing the AI reads. Before any import, before any code — it already knows what the file does, what it depends on, and what it must never break.

Inline Hyperlinks
Micro-Context — Safe for sliding-window agents

AI agents often extract 10–30 lines mid-file, skipping the header entirely. Inline hyperlinks embed navigation cues directly at the function level.

payments.py · lines 48–62 
def apply_discount(base_price: int, tier: str) -> float:
    # @REQUIRES-READ: config.py → MAX_DISCOUNT
    # @REQUIRES-READ: db.py → UserSchema.tier
    # @MODIFIES-ALSO: invoice.py → total()

    if tier == "premium":
        return base_price * 0.8
    return base_price

Specification

The Hyperlink Tags

Three annotation tags define the cross-file navigation protocol for AI agents. They work like hyperlinks on the web — but for LLMs navigating a codebase.

Hyperlink syntax — any language 
# @SEE: file → symbol            — recommended context (read if uncertain)
# @REQUIRES-READ: file → symbol   — MUST read before editing
# @MODIFIES-ALSO: file → symbol   — cascade: update that symbol too
TagSemanticsAgent behavior
@SEE Recommended context Read when uncertain about scope or constraints
@REQUIRES-READ Mandatory prerequisite MUST read the target before writing any logic
@MODIFIES-ALSO Cascade change required MUST update the target symbol as part of this edit

Manifest Header Fields

FieldRequiredRule
FILEYesExact filename — used by orchestrators for manifest-only reads
PURPOSEYes≤15 words describing what, not how
DEPENDS_ONYesfile → func1, func2 or none
EXPORTSYesPublic API with return type — the AI knows what not to rename
STYLEOptionalCSS + chart library, or none
DB_TABLESOptionalTables and relevant columns
LAST_MODIFIEDYes≤8 words — MUST be the first change on every edit

Quick Start

Activate CodeDNA in your AI tool.

Paste one of these into your AI assistant's system prompt. CodeDNA is a pure protocol — no install required, just instructions.

Universal (Claude / GPT-4 / Gemini) 
You follow the CodeDNA v0.3 LLM Navigation Protocol.

EDITING: Read the manifest header first. Follow DEPENDS_ON constraints. Follow every @REQUIRES-READ link before writing logic. Update LAST_MODIFIED as your first change.

GENERATING: Write the manifest header before any imports. Add @REQUIRES-READ/@MODIFIES-ALSO to cross-file functions. Use semantic variable names: list_dict_users_from_db = get_users()

EXPORTS are contracts — never rename without explicit instruction.

Translate Your Code

Before & after CodeDNA

Any file can become a CodeDNA file. The transformation is additive — no code changes, only new information layers.

❌ Standard — no context 
from .utils import get_users, send_email

def notify_inactive(days):
    data = get_users()
    for u in data:
        if u["inactive_days"] > days:
            send_email(u["email"])
# AI reading this has ZERO context:
# What does data contain?
# What happens if send_email fails?
# Does this touch the DB directly?
✅ CodeDNA v0.3 — full context 
# === CODEDNA:0.3 ========================
# FILE: notifications.py
# PURPOSE: Email inactive users after N days
# CONTEXT_BUDGET: normal
# DEPENDS_ON: utils.py → get_users(),
#             utils.py → send_email()
# EXPORTS: notify_inactive(days: int) → None
# DB_TABLES: users (email, inactive_days)
# LAST_MODIFIED: initial implementation
# ========================================
from .utils import get_users, send_email

def notify_inactive(days: int) -> None:
    # @REQUIRES-READ: utils.py → get_users()
    # @MODIFIES-ALSO: logs.py → activity_log()
    list_dict_users_from_db = get_users()
    for user_dict in list_dict_users_from_db:
        if user_dict["inactive_days"] > days:
            send_email(user_dict["email"])
