Il Problema

Gli AI sbagliano quando non hanno contesto.

Immagina di affidare la ristrutturazione di un appartamento a un artigiano, senza mostrargli la planimetria. Stesso problema con gli AI che modificano il tuo codice. CodeDNA è la planimetria.

🔍
Finestra Scorrevole
Gli AI leggono il codice a "finestre" di 50–100 righe. Se una regola critica è definita 200 righe più in alto, non la vedono — e la violano.
❌ Scenario S4: viola MAX_DISCOUNT_RATE
🔗
Effetto Cascata
Cambi una funzione. Altre 3 parti del sistema dipendono da lei. Senza una mappa, l'AI aggiorna solo il file che vede e lascia il resto rotto.
❌ Scenario S5: KeyError a runtime in main.py
🧬
La Soluzione: CodeDNA
Il contesto vive nel file stesso — non in un documento esterno. Ogni frammento che l'AI legge porta con sé le istruzioni per agire correttamente.
✅ Zero errori in tutti i 5 scenari
Come Funziona

Tre livelli. Ogni frammento è autonomo.

Come il DNA biologico: tagliarne metà produce due frammenti che portano ancora l'informazione completa. Con CodeDNA, 10 righe estratte a caso da qualsiasi punto del file bastano all'AI per agire correttamente.

Livello 1
Manifest Header
Le prime 14 righe di ogni file. L'AI le legge prima del codice e conosce già lo scopo del file, le dipendenze, l'API pubblica e i vincoli da rispettare.
pricing.py
# === CODEDNA:0.4 ========================
# FILE: pricing.py
# PURPOSE: Pricing engine with tier discounts
# CONTEXT_BUDGET: always
# DEPENDS_ON: config.py → MAX_DISCOUNT_RATE
# EXPORTS: apply_discount(cents,tier) → int
# REQUIRED_BY: checkout.py → build_cart()
# AGENT_RULES: never exceed MAX_DISCOUNT_RATE
# LAST_MODIFIED: added SUPER20 promo code
# ========================================
Livello 2
Inline Hyperlinks
Tag semantici dentro le funzioni. Anche leggendo solo le righe 200–250 di un file da 500, l'AI trova i link verso ciò che deve sapere prima di agire.
pricing.py · righe 210–222
def apply_discount(cents, tier):
  # @REQUIRES-READ: config.py → MAX_DISCOUNT_RATE
  # @MODIFIES-ALSO: checkout.py → build_cart()
  discount = get_multiplier(tier)
  # cap obbligatorio — da config.py
  discount = min(discount, 0.30)
  return int(cents * (1.0 - discount))
Livello 3
Semantic Naming
I nomi delle variabili codificano tipo, forma e origine. Un frammento da 10 righe è completamente auto-documentante — zero tracciamento a ritroso.

Bonus
Planner Manifest-Only
Per pianificare modifiche su 10+ file, l'AI legge solo le prime 14 righe di ciascuno e usa CONTEXT_BUDGET per filtrare — costruisce la mappa completa in pochi token.
Protocollo Planner
# 1. Leggi solo le prime 14 righe (manifest)
# 2. always → includi sempre
#    normal → includi se rilevante
#    minimal → salta (utility file)
# 3. Costruisci grafo DEPENDS_ON / REQUIRED_BY
# 4. Apri in full SOLO i file necessari
# Costo: ~70 tok × N file = mappa completa
